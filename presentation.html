Agentic Learning Platform

Système d'apprentissage dynamique orchestré par intelligence artificielle. Intégration de modèles multi-agents, architecture de récupération augmentée (RAG), et mécanismes de résilience réseau.

1. Socle Technologique

Interface Utilisateur

  • React & Vite : Architecture SPA optimisée.
  • Tailwind CSS : Typographie et design fluide.
  • Monaco Editor : Interpréteur de code intégré pour le laboratoire interactif (HTML, CSS, JS).

Infrastructure Backend

  • Node.js & Express : Routeur et contrôleurs asynchrones.
  • TypeORM : Gestion relationnelle des entités utilisateurs, plans, et progression.
  • PostgreSQL avec pgvector : Persistance des données et indexation vectorielle.

Moteurs d'Inférence

  • Google Gemini (2.0 Flash) : Modèle primaire pour le raisonnement et la structuration.
  • Groq (Llama 3.3 70b) : Moteur de secours (fallback) ultra-rapide.
  • Embeddings : Conversion textuelle en tenseurs multi-dimensionnels pour la recherche sémantique.

2. Modélisation Mathématique et Moteur RAG

Le mécanisme RAG (Retrieval-Augmented Generation)

Pour garantir la continuité pédagogique, le système mémorise le contexte des cours précédents. Lorsqu'un nouveau module est généré, le backend interroge la base de données pgvector pour extraire les concepts associés, évitant ainsi la redondance et renforçant les liens cognitifs.

Fondation Mathématique : La Similarité Cosinus

Les textes sont transformés en vecteurs dans un espace à haute dimension (ex: 768 dimensions). Pour évaluer la proximité sémantique entre une requête Q et un document D, l'algorithme calcule le produit scalaire normalisé de leurs vecteurs respectifs.

cos(θ) = (Q · D) / (||Q|| × ||D||) = (∑ Q_i D_i) / (√(∑ Q_i²) × √(∑ D_i²))

Implémentation SQL : Dans PostgreSQL, cette opération est exécutée nativement via l'opérateur <=> (distance cosinus). Le système classe les résultats par distance croissante, sélectionnant les k voisins les plus proches pour les injecter dans le prompt du modèle de langage.

3. Orchestration Multi-Agents

La génération du contenu est divisée en processus asynchrones séquentiels. Chaque agent IA reçoit une instruction stricte (System Prompt) et produit un format de données validé avant de le transmettre à l'agent suivant.

Architecte

Définit la taxonomie du cours. Extrait la structure temporelle (phases, semaines) et produit l'arborescence des concepts.

Chercheur

Valide l'exactitude des informations via des appels d'API externes, injectant des références bibliographiques et des liens pertinents.

Rédacteur

Synthétise le contexte RAG et les recherches pour générer des modules d'apprentissage profonds, intégrant du code exécutable et des graphes textuels (Mermaid).

Critique (Assurance Qualité)

Parseur strict garantissant que le résultat final respecte l'interface JSON exigée par le frontend. Rejette ou corrige les hallucinations structurelles.

4. Résilience Réseau et Redondance

Algorithme d'Exponential Backoff

L'interaction avec des modèles LLM distants implique des contraintes de quotas (Rate Limits HTTP 429). Le backend implémente une fonction d'attente exponentielle pour lisser les pics de requêtes.

Temps d'attente = Délai Initial × 2^n

Si l'API primaire refuse la connexion après le nombre maximal de tentatives n, le système bascule de façon transparente vers l'API secondaire (Groq), en lui fournissant un schéma explicite de repli (Fallback Schema) pour maintenir l'intégrité de l'application.

// Pseudocode du Mécanisme de Résilience
function withRetry(operation, maxRetries) {
  let delay = BaseDelay;
  for (let i = 0; i < maxRetries; i++) {
    try { return await operation(); }
    catch (error) {
      if (error.status === 429) { await sleep(delay); delay *= 2; }
    }
  }
  return fallbackOperation();
}

5. Mécanique d'Évaluation et d'Accomplissement

Calcul des Scores

Chaque module inclut des mécanismes d'évaluation dynamique (Quiz et Laboratoire). Le score (XP) est calculé sur le backend et persistant dans la base de données.

Total XP = ∑ (Questions Correctes × Poids) + (Exercices Complétés × Poids_Pratique)

Logique de Certification

La plateforme gère les états de progression par "Phase". Une fois tous les nœuds d'une phase validés, un appel API génère un examen final aléatoire de haut niveau. Un algorithme vérifie que la moyenne dépasse un seuil absolu (T = 0.70) avant de déverrouiller une entité d'Accomplissement (Achievement) dans la base relationnelle.
